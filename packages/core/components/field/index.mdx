# Field

A `Field` wraps a single form field (`<input>, <select>, <textarea>`),
a `name` prop, and optional `hint` and `alert` props into a `<label>` element
and arranges them according to the `direction` and `reverse` props.

Direction is vertical by default. Passing `direction={Direction.HORIZONTAL}`
and `reverse` will position the `label` to the right of the `input`, which is
the preferred layout for a checkbox or radio button.

For a group of radio buttons, use [RadioField](/components/radioField) instead.

### PropTypes

| Prop | Type | Required | Default | Description |
|:-----|:-----|:---------|:--------|:------------|
| Label props | | | | see [Label](/components/label) |
| name | string | yes | | |
| nameWidth | string | no | | applies to direction `HORIZONTAL` only |
| required | bool | no | `false` | appends an asterisk to name |
| direction | enum | no | `VERTICAL` | see `common/direction` |
| reverse | bool | no | `false` | puts name after the input. applies to direction `HORIZONTAL` only |
| desc | string | no | | a description or hint |
| alert | string | no | | bold red message, typically used to signal invalid input |
